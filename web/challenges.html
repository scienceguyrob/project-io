Challenges

A set of applied machine learning challenges designed to guide newcomers through real techniques while providing genuinely tricky problems to solve. Each challenge accompanies a main notebook and uses a realistic dataset from science, engineering, or finance.

Notebook 5 Challenges

5
ChallengeParticle Classification from Detector Signatures
📁 Notebook_5/Challenge_5.ipynb  ·  data: Notebook_5/data/particles.csv

A particle physics classification problem inspired by detectors at CERN. Each sample represents a particle candidate reconstructed from tracker and calorimeter measurements. The task is to classify particles as electron, muon, or pion (20/50/30 split) using eight spectral and kinematic features.

VariableDescription
momentum_gevReconstructed track momentum [GeV/c]
track_chi2Track fit chi-squared per degree of freedom
ecal_energyElectromagnetic calorimeter energy deposit [GeV]
hcal_energyHadronic calorimeter energy deposit [GeV]
ecal_hcal_ratioECAL / (ECAL + HCAL) energy ratio (derived, correlated with both calorimeter features)
dE_dxIonisation energy loss per unit length [MeV/cm] (Bethe-Bloch correlated with momentum)
isolation_scoreCalorimeter cone isolation score
n_hitsNumber of tracker hits along the track
particle_typeTarget label (0=electron, 1=muon, 2=pion)
Goal: Apply multinomial logistic regression, understand why feature scaling is required before gradient-based solvers, and interpret the learned coefficient matrix in the context of detector physics. The electron/pion boundary is the hardest to draw.
5a
ChallengeSea State Classification from Buoy Observations
📁 Notebook_5/Challenge_5a.ipynb  ·  data: Notebook_5/data/sea_state.csv

An entry-level binary classification problem from operational oceanography. Each sample represents a snapshot from a moored ocean buoy. The task is to classify conditions as calm or storm using five meteorological measurements.

VariableDescription
wind_speed_ms10-metre wind speed [m/s]
wave_period_sDominant wave period [seconds]
swell_height_mSignificant swell height [metres]
sst_anomaly_cSea surface temperature anomaly [°C] (weak predictor)
pressure_hpaMean sea level atmospheric pressure [hPa] (negatively correlated with wind and wave features)
sea_stateTarget label (0=calm, 1=storm)
Goal: Apply binary logistic regression, understand why scaling is required, interpret the learned coefficients physically (wind and wave features positive; pressure negative), and recognise that sst_anomaly_c contributes little signal — which students should predict from the KDE plots before fitting and confirm from the coefficient magnitude.

Notebook 6 Challenges

6a
ChallengeLand Cover Classification from Satellite Imagery
📁 Notebook_6/Challenge_6a.ipynb  ·  data: Notebook_6/data/robsat.csv

A remote sensing classification problem inspired by Sentinel-2 multispectral imagery. Each sample represents a single pixel observation described by spectral reflectance bands and derived indices. The task is to classify pixels as urban, vegetation, or water (60/30/10 split) using eight features.

VariableDescription
blue_refBlue band reflectance (~490 nm)
red_refRed band reflectance (~665 nm)
nir_refNear-infrared reflectance (~842 nm)
swir_refShort-wave infrared reflectance (~1610 nm)
ndviNormalised Difference Vegetation Index (derived from NIR and red)
ndwiNormalised Difference Water Index (derived; anti-correlated with NDVI at ~−0.96)
texture_varLocal variance of NIR in a 3×3 pixel window
thermal_btThermal infrared brightness temperature [K]
land_coverTarget label (0=urban, 1=vegetation, 2=water)
Goal: Apply a decision tree classifier, use the elbow method and silhouette scores to diagnose the right tree depth, visualise the fitted tree, and interpret feature importances. The unconstrained tree achieves 100% training accuracy and 91% test accuracy; optimal depth=3 achieves 95.8% — making the bias-variance tradeoff visible and measurable.
6b
ChallengeGas Turbine Operational State Classification
📁 Notebook_6/Challenge_6b.ipynb  ·  data: Notebook_6/data/temp_monitoring.csv

A safety-critical classification problem set in an industrial gas turbine monitoring context. Each sample is a telemetry snapshot from a heavy-duty power generation turbine. The task is to classify the operational state as normal, warning, critical, or fault (50/25/20/5 split) using seven temperature sensor readings. The dataset contains realistic missing values and spike outliers.

VariableDescription
egt_cExhaust gas temperature [°C] (most reliable sensor, rarely missing)
cit_cCompressor inlet temperature [°C]
tbmt_cTurbine blade metal temperature [°C] (~12.5% missing — optical pyrometer signal loss)
brg_cBearing temperature [°C] (~3% missing — vibration-induced dropout)
ics_cInter-stage cooling supply temperature [°C] (~8.8% missing, conditional on tbmt_c)
lube_cLube oil temperature [°C] (correlated with brg_c)
amb_cAmbient temperature [°C]
op_stateTarget label (0=normal, 1=warning, 2=critical, 3=fault)
Goal: Apply Gaussian Naive Bayes after handling missing values and outliers without data leakage. Students must cap outliers using IQR bounds fitted on training data, then apply median imputation using SimpleImputer. The fault class has a bimodal distribution (mechanical runaway vs sensor failure) that a single Gaussian cannot represent — this produces high variance in the fitted fault parameters and explains the lower fault recall.

Notebook 7 Challenges

7a
ChallengeVariable Star Classification from Light Curve Spectra
📁 Notebook_7/Challenge_7a.ipynb  ·  data: Notebook_7/data/spectra.csv

An astrophysical classification problem inspired by large photometric surveys such as OGLE and Gaia. Each sample represents a variable star candidate described by pulsation statistics and photometric measurements. The task is to classify stars into seven types (Delta Scuti, RR Lyrae, Cepheid, Mira, W Virginis, Beta Cephei, SX Phoenicis) with a 28/22/18/12/9/7/4 distribution.

VariableDescription
period_minDominant pulsation period [minutes] (spans ~4 orders of magnitude)
amplitude_magPeak-to-peak V-band amplitude [magnitudes]
bp_rpGaia BP-RP colour index [mag] (effective temperature proxy)
phi21Fourier phase parameter φ₂₁ from light curve decomposition [radians]
r21Fourier amplitude ratio R₂₁ = A₂/A₁
fe_h_proxyIron abundance [Fe/H] proxy [dex] (primary Population I / II discriminator)
skewnessPhased light curve skewness
luminosity_lsunLuminosity in solar units (spans ~5 orders of magnitude)
n_modeslog₁₀ of number of detected pulsation frequencies
star_typeTarget label (0–6)
Goal: Apply an RBF SVM and demonstrate the catastrophic effect of not scaling features. period_min and luminosity_lsun span orders of magnitude, causing the unscaled SVM to achieve only 23–56% accuracy while the scaled SVM reaches 79–83% — a ~34 percentage point gain from scaling alone.
7b
ChallengeNetwork Intrusion Detection with Random Forests
📁 Notebook_7/Challenge_7b.ipynb  ·  data: Notebook_7/data/security.csv

A cybersecurity classification problem inspired by the NSL-KDD network intrusion benchmark. Each sample represents a labelled network connection record. The task is to classify traffic as normal, DoS, probe, or R2L (35/24/26/15 split) using 15 features including 10 base network statistics and 5 noisy derived scores.

VariableDescription
duration_sConnection duration [seconds]
packet_ratePackets per second
byte_rateBytes per second (correlated with packet_rate)
syn_flag_rateFraction of SYN-flagged packets
error_rateFraction of RST/error packets
failed_loginsFailed authentication attempts (primary R2L discriminator)
payload_entropyShannon entropy of payload bytes [bits]
traffic_typeTarget label (0=normal, 1=dos, 2=probe, 3=r2l)
Goal: Apply RandomForestClassifier and use GridSearchCV with stratified cross-validation to tune n_estimators, max_depth, max_features, and min_samples_leaf, scoring on f1_macro. The R2L class deliberately mimics normal traffic; a weak default configuration achieves only 41% R2L recall, which a tuned model improves to 70%.

Notebook 8 Challenges

8a
ChallengeGalaxy Clustering and Semi-Supervised Classification
📁 Notebook_8/Challenge_8a.ipynb  ·  data: Notebook_8/data/galaxies.csv & unknown_galaxies.csv

An extragalactic astrophysics challenge inspired by SDSS and GAMA galaxy surveys. The labelled dataset contains 10,000 galaxy observations; the unlabelled dataset contains 2,000 galaxies with no class column. The three galaxy types are star-forming, quiescent, and starburst (45/35/20 split).

VariableDescription
log_sfrlog₁₀ star formation rate [M☉/yr] (primary discriminator)
log_stellar_masslog₁₀ stellar mass [M☉]
u_r_colourRest-frame (u−r) colour index [magnitudes]
dn40004000 Å break strength (correlated with u_r_colour at ~0.93)
h_alpha_ewHα equivalent width [Å] (strong in starburst)
velocity_dispersionStellar velocity dispersion [km/s]
galaxy_typeTarget label in galaxies.csv only (0=star_forming, 1=quiescent, 2=starburst)
Goal: A three-part challenge: (1) apply K-Means clustering with elbow method and silhouette scores, evaluate with Adjusted Rand Index; (2) use 50 labels per class as seeds with KNN to classify the 2,000 unlabelled galaxies; (3) fit a Gaussian Mixture Model and compare to K-Means — diag covariance GMM achieves the best cluster recovery (ARI=0.913) while full covariance overfits despite lower BIC.

Notebook 9 Challenges

9
ChallengeDimensionality Reduction and MLP: Pulsar Candidate Classification
📁 Notebook_9/Challenge_9.ipynb  ·  data: Notebook_9/data/radio.csv

A radio astronomy classification problem directly inspired by the HTRU-2 pulsar survey dataset (Lyon et al. 2016). Each sample represents a pulsar candidate described by statistics of its folded pulse profile and DM-SNR curve. The task is binary: non-pulsar (90%) or pulsar (10%). The dataset has 20 features: the 8 original HTRU-2 features plus 12 correlated noisy extensions simulating a modern survey pipeline.

VariableDescription
prof_mean, prof_std, prof_skewness, prof_kurtosisStatistics of the integrated folded pulse profile
dm_snr_mean, dm_snr_std, dm_snr_skewness, dm_snr_kurtosisStatistics of the DM-SNR curve
subband_01–subband_08Sub-band profile and DM statistics (noisy correlated copies)
pipeline_01–pipeline_04Higher-order pipeline diagnostic scores
candidate_typeTarget label (0=non_pulsar, 1=pulsar)
Goal: Walk through a complete PCA pipeline: diagnose the correlation structure, apply StandardScaler, produce a scree plot and cumulative explained variance curve, visualise the data in 2D PCA space, inspect component loadings, then train an MLP on PCA-reduced features and compare to the full 20-feature baseline. The key finding is that 2 PCA components explain 28% of variance but produce 98.0% MLP accuracy — nearly matching the 98.2% from all 20 features.

10
ChallengeK-NN and SMOTE: Satellite Telemetry Anomaly Detection
📁 Notebook_10/Challenge_10.ipynb  ·  data: Notebook_10/data/satellite.csv

A safety-critical anomaly detection problem set in an Earth-observation satellite ground control context (inspired by ESA Sentinel operations). Each sample is a one-minute telemetry snapshot. The task is to classify the satellite's operational state as nominal, thermal anomaly, power anomaly, or attitude anomaly (59/18/14/9 split).

VariableDescription
battery_voltage_vBattery terminal voltage [V]
solar_panel_current_aSolar array output current [A] (drops to ~0 in eclipse)
cpu_temp_cOnboard processor temperature [°C]
reaction_wheel_rpmReaction wheel spin rate [rpm] (elevated in attitude anomaly)
attitude_error_roll_degRoll pointing error [degrees]
attitude_error_pitch_degPitch pointing error [degrees]
attitude_error_yaw_degYaw pointing error [degrees]
link_quality_dbDownlink signal quality [dBm relative]
power_consumption_wTotal satellite power consumption [W]
eclipse_flagBinary: 1 = satellite in Earth's shadow
operational_stateTarget label (0=nominal, 1=thermal_anomaly, 2=power_anomaly, 3=attitude_anomaly)
Goal: A three-part challenge demonstrating the full imbalanced learning workflow. (1) Establish a baseline KNeighborsClassifier and show that attitude anomaly recall is only 64.6%. (2) Apply SMOTE to the scaled training set only, showing a +17 percentage point jump in attitude recall. (3) Run GridSearchCV with f1_macro scoring to tune k, distance weighting, and metric. Also introduces Matthews Correlation Coefficient (MCC) as a more informative single-number metric than accuracy for imbalanced problems.

ChallengeSVM Overfitting and Regularisation: Solar Flare Classification
📁 Notebook_11/Challenge_11.ipynb  ·  data: Notebook_11/data/sol.csv

A space weather classification problem inspired by NASA SDO and NOAA GOES solar monitoring data. Each sample represents a snapshot of solar observatory measurements. The task is to classify solar activity as quiet, active, or eruptive (44/36/20 split) using 12 features: six informative spectroscopic and magnetic measurements plus six noisy derived spectral ratio features.

VariableDescription
log_xray_fluxlog₁₀ GOES 1–8 Å X-ray flux [W/m²] (primary flare discriminator)
halpha_intensityRelative H-alpha chromospheric emission
magnetic_bmaxPeak active region magnetic field [Gauss]
sunspot_areaSunspot group area [millionths of solar disk]
prominence_heightTallest prominence height above limb [Mm]
radio_burst_sfuType II/III radio burst flux density [SFU]
spectral_ratio_1–spectral_ratio_3Spectral line ratios (noisy; spectral_ratio_3 is near-pure noise)
activity_classTarget label (0=quiet, 1=active, 2=eruptive)
Goal: Demonstrate and diagnose SVM overfitting through systematic C and gamma sweeps. The key finding: at gamma=10 the SVM achieves 100% training accuracy but only 44% test accuracy, a 56 percentage point gap. Students produce C-sweep curves, gamma-sweep curves, and a 2D heatmap across a C-gamma grid. GridSearchCV finds the well-regularised configuration: overfitting SVM (5,670 support vectors, 10pp gap) vs tuned SVM (1,554 support vectors, ~1pp gap).

ChallengeComparing SVM, Random Forest, and MLP: Stock Market Movement Prediction
📁 Notebook_12/Challenge_12.ipynb  ·  data: Notebook_12/data/market.csv

A quantitative finance classification problem using technical indicators and market context features. Each sample represents one trading day's end-of-day measurements for a stock. The task is to predict next-day price movement direction: down (>0.5% fall), flat (within ±0.5%), or up (>0.5% rise), with a 29/38/33 distribution.

VariableDescription
rsi_1414-day Relative Strength Index [0–100]
macd_signalMACD signal line value
bb_positionBollinger Band position: (price − lower) / (upper − lower) [0–1]
volume_ratioToday's volume / 20-day average
atr_pctAverage True Range as % of price (volatility measure)
price_momentum_5d5-day price return [%]
ma_cross(5-day MA / 20-day MA) − 1 (trend signal)
overnight_gap(open − prior close) / prior close [%]
market_sentimentComposite sentiment index [−1 to 1]
sector_betaStock beta relative to sector index
earnings_proximityNormalised days to nearest earnings announcement [−1 to 1]
spread_proxyBid-ask spread proxy as % of price
movementTarget label (0=down, 1=flat, 2=up)
Goal: A full multi-classifier comparison following a rigorous no-leakage protocol. All three models are wrapped in Pipeline objects with StandardScaler, ensuring preprocessing parameters are estimated inside each cross-validation fold. Steps through defining pipelines, running 5-fold stratified cross-validation with cross_validate, training final models, evaluating on the sealed test set, plotting confusion matrices, and producing one-vs-rest ROC curves. Students make a reasoned judgement about which model to deploy and why.
