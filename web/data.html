Datasets

A reference for every dataset used across the project-io notebook series and challenge files — including file paths, column descriptions, class distributions, and usage notes.

Notebook 4

Notebook 4Universities.csv
📁 Notebook_4/data/Universities.csv  ·  207 data rows (208 including header)  ·  Real dataset

A real dataset of US university programme completion statistics. Used throughout Notebook 4 as the primary tabular data example for pandas.

ColumnTypeDescription
SectorString (categorical)The sector of the institution (e.g. "Private for-profit, 2-year")
UniversityStringName of the institution
YearIntegerThe academic year of the record
CompletionsIntegerNumber of programme completions reported
GeographyString (categorical)US state in which the institution is located
Note on CSV format: Values containing commas (e.g. "Private for-profit, 2-year") are enclosed in speech marks so the CSV reader treats the entire text as a single field rather than splitting at the comma.

Notebook 6

Notebook 6clouds.csv
📁 Notebook_6/data/clouds.csv  ·  1,800 rows  ·  Synthetic  ·  3 classes, 600 per class

Synthetic cloud observations described by four continuous physical measurements. Used in Notebook 6 Section 6 to introduce decision trees. Deliberate overlap between Cumulus and Stratus (both occur at low altitudes with mild temperatures) means the tree must use multiple features to separate them.

FeatureUnitsDescription
altitude_kmkmHeight of the cloud base above sea level. Cirrus forms at 6–12 km; stratus and cumulus below 3 km.
vertical_extent_kmkmThickness of the cloud layer from base to top.
temperature_c°CAir temperature at cloud altitude. Drops ~6.5°C per km of altitude.
optical_thickness0–10How opaque the cloud appears. Thick stratus blocks most sunlight; wispy cirrus is nearly transparent.
cloud_type—Target label: Cumulus, Stratus, or Cirrus.
Notebook 6clouds_complex.csv
📁 Notebook_6/data/clouds_complex.csv  ·  6,000 rows  ·  Synthetic  ·  3 classes, 2,000 per class

A more challenging synthetic cloud dataset with ten features, non-linear relationships, correlated noise, and approximately 5% deliberate label noise. Used in Notebook 6 Section 7.3 to compare a single decision tree against a random forest. This dataset must be generated before any cell that loads it.

Sources of difficulty include: wider class distributions so Cumulus and Stratus overlap substantially; non-linear feature relationships that a single decision tree struggles to capture; correlated measurement noise across features; and deliberate mislabelling of ~5% of observations.

FeatureUnitsDescription
altitude_kmkmCloud base height above sea level.
vertical_extent_kmkmCloud layer thickness from base to top.
temperature_c°CAir temperature at cloud altitude.
optical_thickness0–12Opacity to incoming sunlight.
droplet_radius_umμmEffective radius of water droplets or ice crystals. U-shaped relationship with altitude.
liquid_water_pathg/m²Total liquid water mass per vertical column. Non-linear interaction with vertical extent.
wind_shearm/s/kmChange in wind speed across the cloud layer. U-shaped with altitude.
humidity_pct%Relative humidity at cloud altitude. Stratus ~88%; Cirrus ~35%.
albedo0–1Fraction of sunlight reflected. Non-linear function of optical thickness and liquid water path.
pressure_hpahPaAtmospheric pressure at cloud base. Decreases exponentially with altitude.
cloud_type—Target label: Cumulus, Stratus, or Cirrus.
Notebook 6credit.csv
📁 Notebook_6/data/credit.csv  ·  5,000 rows  ·  Synthetic  ·  Binary target

Synthetic credit application records. Each row represents a single applicant described by six financial features and a binary risk label. Used in Notebook 6 to demonstrate SVM classification with feature scaling — note that credit_score runs from 300–850 while debt_ratio sits between 0 and 1, which is precisely why scaling is essential before fitting an SVM.

FeatureDescription
incomeAnnual income (£k)
debt_ratioProportion of income committed to existing debt
credit_scoreCredit score (300–850)
loan_amountLoan amount requested (£k)
employment_yearsYears in current employment
missed_paymentsNumber of missed payments in the past 12 months
riskTarget label: 0 = Low risk, 1 = High risk
Challenge 6arobsat.csv
📁 Notebook_6/data/robsat.csv  ·  Synthetic  ·  3 classes (60/30/10 split)

Remote sensing data inspired by Sentinel-2 multispectral imagery. Each row is a single pixel observation with spectral reflectance bands and derived indices. The target is land cover type: urban, vegetation, or water. See the Challenges page for the full feature list.

Challenge 6btemp_monitoring.csv
📁 Notebook_6/data/temp_monitoring.csv  ·  Synthetic  ·  4 classes (50/25/20/5 split)  ·  Contains missing values

Industrial gas turbine telemetry with seven temperature sensor readings. Includes realistic missing values (~12.5% in tbmt_c, ~8.8% in ics_c, ~3% in brg_c) and spike outliers. Target is operational state: normal, warning, critical, or fault. See the Challenges page for the full feature list.

Notebook 7

Built-inBreast Cancer Wisconsin (Diagnostic) Dataset
Loaded via sklearn.datasets.load_breast_cancer()  ·  569 rows  ·  30 features  ·  Binary target  ·  Used in Notebooks 7 & 13

Each row represents a tumour biopsy. The 30 numeric features are computed from a digitised image of a needle biopsy sample and are organised into three groups of 10 (mean, standard error, and worst) describing characteristics of cell nuclei.

The target variable is binary: 0 = Malignant, 1 = Benign. The dataset is class-imbalanced: approximately 63% Benign and 37% Malignant.

Used in Notebook 7 for visualisation and feature selection examples. Also used in Notebook 13 as the primary benchmark for comparing bagging ensembles and AdaBoost — a 100-tree bagging ensemble and a 200-stump AdaBoost model are both trained and evaluated on this dataset.

Feature groupExamplesHow computed
meanmean radius, mean textureMean value across all nuclei in the image
se (standard error)radius error, texture errorStandard error of the measurement
worstworst radius, worst textureMean of the three largest values observed

The 30 features span: radius, texture, perimeter, area, smoothness, compactness, concavity, concave points, symmetry, and fractal dimension — each measured three ways (mean, SE, worst).

Challenge 7aspectra.csv
📁 Notebook_7/data/spectra.csv  ·  Synthetic  ·  7 classes (28/22/18/12/9/7/4 distribution)

Astrophysical variable star data inspired by OGLE and Gaia surveys. Contains pulsation statistics and photometric measurements. Key challenge: period_min and luminosity_lsun each span ~4–5 orders of magnitude, making feature scaling critical. See the Challenges page for the full feature list.

Challenge 7bsecurity.csv
📁 Notebook_7/data/security.csv  ·  Synthetic  ·  4 classes (35/24/26/15 split)

Network intrusion detection data inspired by the NSL-KDD benchmark. Labelled network connection records with 15 features (10 base statistics + 5 noisy derived scores). Target: normal, DoS, probe, or R2L traffic. The R2L class deliberately mimics normal traffic. See the Challenges page for the full feature list.

Notebook 8

Challenge 8agalaxies.csv & unknown_galaxies.csv
📁 Notebook_8/data/galaxies.csv (10,000 rows, labelled)  ·  Notebook_8/data/unknown_galaxies.csv (2,000 rows, unlabelled)

Extragalactic astrophysics data inspired by SDSS and GAMA galaxy surveys. Both files share the same 8 features. galaxies.csv has labels; unknown_galaxies.csv does not. Three galaxy types: star-forming, quiescent, and starburst (45/35/20 split). See the Challenges page for the full feature list.

Notebook 9

Built-inIris Dataset
Loaded via sklearn.datasets.load_iris()  ·  150 rows  ·  4 features  ·  3 classes, 50 per class

The classic Iris flower dataset. Used in Notebook 9 to demonstrate PCA: iris.data is a 150×4 matrix where each row is one flower and each column is one of four measurements. After standardising, PCA(n_components=2) reduces to 2 principal components.

ColumnFeatureMeaning
0sepal length (cm)Length of the sepal (the leaf-like part beneath the petals)
1sepal width (cm)Width of the sepal
2petal length (cm)Length of the petal
3petal width (cm)Width of the petal
Challenge 9radio.csv
📁 Notebook_9/data/radio.csv  ·  Synthetic  ·  Binary target (90% non-pulsar / 10% pulsar)  ·  20 features

Radio astronomy pulsar candidate classification data directly inspired by the HTRU-2 survey dataset (Lyon et al. 2016). Contains 8 original HTRU-2 features plus 12 correlated noisy extensions simulating a modern survey pipeline. See the Challenges page for the full feature list.

Notebook 10

Challenge 10satellite.csv
📁 Notebook_10/data/satellite.csv  ·  Synthetic  ·  4 classes (59/18/14/9 split)

Earth-observation satellite telemetry inspired by ESA Sentinel operations. Each row is a one-minute snapshot with 10 sensor readings. Target is operational state: nominal, thermal anomaly, power anomaly, or attitude anomaly. The imbalanced class distribution makes this a good test case for SMOTE and MCC-based evaluation. See the Challenges page for the full feature list.

Notebook 11

Challenge 11sol.csv
📁 Notebook_11/data/sol.csv  ·  Synthetic  ·  3 classes (44/36/20 split)  ·  12 features

Space weather classification data inspired by NASA SDO and NOAA GOES solar monitoring. Contains 6 informative spectroscopic and magnetic features plus 6 noisy derived spectral ratios. Target is solar activity class: quiet, active, or eruptive. Designed to demonstrate SVM overfitting and regularisation via C and gamma sweeps. See the Challenges page for the full feature list.

Notebook 12

Challenge 12market.csv
📁 Notebook_12/data/market.csv  ·  Synthetic  ·  3 classes (29/38/33 split)  ·  12 features

Stock market technical indicator data. Each row represents one trading day's end-of-day measurements. Target is next-day price movement direction: down (>0.5% fall), flat (within ±0.5%), or up (>0.5% rise). Used to compare SVM, Random Forest, and MLP in a rigorous no-leakage pipeline. See the Challenges page for the full feature list.

Challenge-Only Datasets (Notebook 5)

Challenge 5particles.csv
📁 Notebook_5/data/particles.csv  ·  Synthetic  ·  3 classes (20/50/30 split)  ·  9 features

Particle physics classification data inspired by CERN detector signatures. Each sample is a particle candidate with tracker and calorimeter measurements. Target: electron (0), muon (1), or pion (2). The electron/pion boundary is the hardest to draw. See the Challenges page for the full feature list.

Challenge 5asea_state.csv
📁 Notebook_5/data/sea_state.csv  ·  Synthetic  ·  Binary target  ·  6 features

Operational oceanography buoy measurements. Binary classification of sea state as calm (0) or storm (1). The sst_anomaly_c feature contributes little predictive signal, which students should identify from KDE plots before fitting. See the Challenges page for the full feature list.
